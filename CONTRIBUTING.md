# Contributing to Kaspathon Developer Resources

Thank you for your interest in contributing! This repository is designed to help hackathon participants learn and build on Kaspa.

## How to Contribute

### Reporting Issues
- Check if the issue already exists
- Provide clear description and steps to reproduce
- Include relevant code snippets or screenshots

### Adding Content

**Tool Listings:**
1. Add to `/tools-and-sdks/developer-tools.md`
2. Include description, repository link, and use cases
3. Categorize appropriately

**Showcase Your Project:**

Built something cool during Kaspathon? Share it with the community!

1. Add your project to `/existing-projects/showcase.md`
2. Include:
   - **Project Name** and type (wallet, DeFi, tool, etc.)
   - **Website/Repository URL**
   - **Description** (2-3 sentences about what it does)
   - **Key Features** (3-5 bullet points)
   - **Use Case** (who should use it and why)
   - **Tech Stack** (languages/frameworks used)
3. Follow the existing format in the showcase file
4. Place in the appropriate category (Wallets, DeFi & Tokens, etc.)
5. Share in Discord `#show-and-tell` channel

Your project will inspire other developers and showcase the Kaspa ecosystem!

### Submitting Changes

1. Fork this repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Test thoroughly
5. Commit with clear messages
6. Push to your fork
7. Create a Pull Request

### Style Guidelines

**Markdown:**
- Use clear headings and structure
- Include code blocks with language tags
- Add links to external resources
- Keep lines reasonably short

**Code:**
- Follow JavaScript/TypeScript standard style
- Comment complex logic
- Include error handling
- Provide usage examples

## Questions?

Ask in the [Kaspathon Discord channel](https://discord.com/channels/599153230659846165/1451212885790560417) or open a GitHub issue.

Thank you for helping make Kaspa development more accessible!
